## 1. MD-индикатор и переключатель в главном окне

### Идея
Когда текст в overlay содержит Markdown (результат сценария с MD-форматированием),
главное окно показывает визуальный сигнал и позволяет переключать отображение.

### Реализация

**Детектор MD:** функция `_is_markdown(text) -> bool`
- Проверяем наличие: `##`, `**`, `- [ ]`, `` ` ``, `---`, `|` (таблицы)
- Порог: ≥2 маркера → считаем MD

**Индикатор:**
- Вокруг TerminalTextView (основное текстовое поле) — тонкая рамка цвета `C_GREEN_BORD`
  (чуть ярче обычной, ~0.3 alpha green) когда текст = MD
- В правом верхнем углу рамки — метка `[MD]` (NSButton/label, кликабельная)
- Без MD — рамка обычная (как сейчас), `[MD]` скрыта

**Состояние:**
```python
_st["md_mode"]    = False   # True = показываем rendered MD, False = raw
_st["is_md"]      = False   # True = текст определён как Markdown
```

**Переключатель [MD]:**
- Клик по `[MD]` → `_st["md_mode"] = not _st["md_mode"]`
- Raw mode: TerminalTextView показывает текст как есть (сейчас)
- Rendered mode: заменяем текст в TerminalTextView на "красивый" вид
  - Заголовки `##` → UPPER CASE + подчёркивание `─────`
  - `**bold**` → `*bold*` (или CAPS в терминальном стиле)
  - `- [ ]` → `○ `, `- [x]` → `✓ `
  - `` `code` `` → оставляем как есть (моноширинный)
  - Блоки кода → рамочка из `│`

  (Terminal-style renderer — без WebKit, чистый Python/NSAttributedString)

## 2. Две кнопки вставки

### Текущее поведение
- `Shift+Enter` → вставить текст как есть (paste)

### Новое поведение
| Комбинация | Действие |
|---|---|
| `Shift+Enter` | Вставить **raw text** (без MD-разметки, чистый текст) |
| `Alt+Shift+Enter` (`⌥⇧↵`) | Вставить **MD-текст** как есть (с `##`, `**` и т.д.) |

**Реализация:**
- В `TerminalTextView.keyDown_`: добавить проверку `Alt+Shift+Enter` (keyCode 36 + Shift + Option)
- `_on_paste_cb` принимает параметр `mode: str = "raw"` → `"raw"` | `"md"`
- В `main.py` → `_on_paste(mode)`: если `mode == "raw"` → strip MD markers перед вставкой
- Простой `strip_markdown(text)` → убираем `#`, `**`, `- [ ]`, `` ` ``, etc.
