HUSH — это инструмент для тех, кто думает быстрее, чем печатает. Зажал хоткей, сказал вслух, отпустил — текст уже в нужном приложении. Опционально: прогнал через LLM, получил готовое письмо, список задач или отформатированную заметку. Распознавание речи работает полностью на устройстве через Apple Neural Engine — никакой телеметрии, никаких облачных микрофонов.

## Почему HUSH?

Идея возникла после знакомства с приложением [Spokenly](https://spokenly.app). Оно показало, что голосовой ввод с постобработкой через LLM — это действительно удобно. Но захотелось своего: без платных ограничений, без перегруженного интерфейса, под конкретные рабочие сценарии.

HUSH создавался и дорабатывался в боевом режиме — я пользуюсь им каждый день. Поэтому каждая деталь появилась из реальной потребности, а не ради галочки. Надеюсь, что он окажется удобным и для вас.

Всё можно настроить под себя: сценарии, провайдеры, темы. Никаких платных версий — только открытый код.

## Как это работает

HUSH — это приложение-аксессуар: нет иконки в Dock, только иконка в строке меню. Запустили — забыли о нём. Оно всегда готово принять голос в любом приложении.

**Под капотом:**

- Речь захватывается через `sounddevice` прямо в RAM
- Распознавание — NVIDIA Parakeet TDT 0.6B, скомпилированный под CoreML / Apple Neural Engine (~400 МБ, входит в поставку)
- При необходимости текст отправляется в LLM (локальный Ollama или облачный Anthropic / OpenAI / GLM)
- Результат вставляется в активное приложение через Accessibility API — без буфера обмена, без мигания

Первый запуск компилирует CoreML-модель под ваш конкретный чип. Это занимает до минуты и происходит один раз. Все последующие запуски — мгновенные.

## Режимы работы

### Тихий режим — Right ⌥

Самый быстрый способ диктовать. Не требует никаких дополнительных действий.

1. **Зажать** Right ⌥ → у края экрана появляется небольшой плавающий индикатор записи
2. **Говорить** — индикатор показывает, что идёт запись
3. **Отпустить** → фрагмент транскрибируется

Если нужно надиктовать несколько кусков:

- Зажать–отпустить несколько раз → фрагменты **накапливаются**
- Когда пауза длится **4 секунды** — начинается обработка через LLM
- Столбики индикатора визуализируют обратный отсчёт: зеленеют → краснеют
- **Enter** во время отсчёта → вставить немедленно без LLM
- **Hover** во время обработки → появляется кнопка прерывания (вставить сырой транскрипт)

Индикатор можно перетащить в любое место экрана — позиция сохраняется между сессиями.

---

### Полный режим — ⇧⌥

Для сложных задач: диктовка по частям, выбор сценария, просмотр результата перед вставкой.

1. **⇧⌥** → открывается большое окно (без немедленного начала записи)
2. **Зажать ⌥** → записать фрагмент → **отпустить** → транскрипт появляется в окне как отдельный блок
3. Повторять сколько угодно — блоки накапливаются в окне
4. Выбрать нужный сценарий и нажать его кнопку — текст уйдёт в LLM
5. **Shift+Enter** → вставить результат в предыдущее активное приложение, окно закрывается

Если в настройках задан сценарий «по умолчанию для полного режима» (★), он применяется автоматически при Shift+Enter — можно не нажимать отдельную кнопку.

---

### Расширенный режим — двойной клик по заголовку

Когда надиктовано много текста и его нужно спокойно прочитать, отредактировать или просто удобно видеть — главное окно можно развернуть.

- **Двойной клик** в любом месте заголовка главного окна → окно разворачивается до 640×680
- Повторный двойной клик → возврат к компактному размеру

В расширенном режиме вспомогательные панели (настройки, история, провайдеры, редактор) автоматически собираются в кластер 2×2 рядом с окном:

- Перетащить любую панель кластера → **весь кластер двигается вместе**
- **🔄** в настройках → сбросить кластер в позицию по умолчанию
- **🎯** → показать / скрыть все панели кластера

## Сценарии постобработки

Сценарии — это настраиваемые системные промпты, которые LLM применяет к расшифрованному тексту. Каждый сценарий — это кнопка в интерфейсе. Можно редактировать встроенные и создавать свои.

Конфигурация хранится в `~/.config/hush/scenarios.json`.

### Встроенные сценарии

| Сценарий | Что делает |
|----------|------------|
| **MAIN** | Умный форматировщик: определяет тип текста (промпт, задачи, письмо, заметка) и применяет соответствующее форматирование; убирает слова-паразиты и оговорки |
| **ЧИСТКА** | Расставляет пунктуацию, заглавные буквы, типографские кавычки («»); чистит паразиты без изменения смысла |
| **ПИСЬМО** | Оформляет текст как деловое письмо: обращение, структура, подпись |
| **Задачи** | Преобразует поток мыслей в список задач с чекбоксами в формате Markdown |
| **MD** | Форматирует как Markdown с заголовками, списками и блоками кода |

> Сценарий без промпта вставляет сырой транскрипт — удобно как «кнопка быстрой вставки».

### Флаги сценариев

- **`тихий режим`** — назначить этот сценарий по умолчанию для тихого режима (только один)
- **`по умолчанию ★`** — назначить этот сценарий по умолчанию для полного режима (только один)

## Редактор сценариев

Открыть: ⚙ Настройки → нажать на любой сценарий.

| Поле                        | Описание                                                                     |
| --------------------------- | ---------------------------------------------------------------------------- |
| **Название** (RU / EN / ES) | До 6 символов — отображается на кнопке сценария                              |
| **Модель**                  | Переопределить LLM для этого конкретного сценария (пусто = авто)             |
| **Промпт**                  | Системная инструкция; расшифрованный текст добавляется в конец автоматически |
| **Тихий режим**             | Назначить сценарий по умолчанию для тихого режима                            |
| **По умолчанию ★**          | Назначить сценарий по умолчанию для полного режима                           |

При переключении между сценариями с несохранёнными изменениями HUSH спросит: **Сохранить / Отменить**.

## История транскрипций

Последние **50 транскрипций** сохраняются автоматически в `~/.config/hush/history.json`.

Панель истории (🕐) содержит три вкладки:

| Вкладка | Содержание |
|---------|------------|
| **Все** | Все блоки в хронологическом порядке |
| **Сессии** | Группировка: несколько фрагментов одной сессии объединены в одну запись |
| **Блоки** | Только отдельные фрагменты |

**Что можно сделать с историей:**

- Клик по элементу → добавить его как новый блок в текущую сессию полного режима
- Чекбоксы → множественный выбор
- Кнопки при множественном выборе: **Удалить** / **Объединить** / **Добавить** / **Заменить**

Панель остаётся открытой после вставки — можно последовательно выбирать и комбинировать несколько записей.

## Провайдеры LLM

Настроить: ⚙ → **[КЛЮЧИ]**.

| Провайдер | Тип | Что нужно |
|-----------|-----|-----------|
| **Ollama** | Локально | Установленный [Ollama](https://ollama.ai) + модель (`ollama pull <model>`) |
| **Anthropic** | Облако | API ключ (`sk-ant-...`) |
| **OpenAI** | Облако | API ключ (совместим с любым OpenAI-совместимым API) |
| **GLM** | Облако | API ключ Zhipu GLM-4 |

### Выбор модели в сценарии

Модель для каждого сценария выбирается в редакторе сценариев через два выпадающих списка:

1. **Провайдер** — выбрать из настроенных и доступных (Ollama / Anthropic / OpenAI / GLM)
2. **Модель** — список автоматически заполняется моделями, доступными у выбранного провайдера

Если провайдер не настроен или недоступен — он не появится в списке. Если модель не выбрана, HUSH использует авто-стратегию: сначала пробует Ollama, при недоступности — Anthropic.

## Установка

### Требования

- **macOS 13+** (Ventura или новее)
- **Python 3.14** — `brew install python@3.14`
- **Разрешение Accessibility** — для автоматической вставки текста (запрашивается при первом запуске)
- **Разрешение Microphone** — запрашивается при первом запуске

Опционально, для LLM-сценариев:
- [Ollama](https://ollama.ai) с загруженной моделью
- API ключи Anthropic / OpenAI / GLM

### Шаги

```bash
git clone https://github.com/alexbic/hush.git
cd hush

# Установить зависимости
pip3.14 install pyobjc sounddevice pynput anthropic openai

# Собрать app bundle
bash build_app.sh

# Запустить
open HUSH.app
```

При первом запуске HUSH автоматически:
1. Копирует `parakeet-cli` в `~/.local/bin/`
2. Размещает CoreML-модель в `~/.local/share/hush/`
3. Компилирует модель под ваш чип (ANE) — происходит один раз, кеш сохраняется между обновлениями

После этого HUSH появится в строке меню и будет готов к работе.

## Приватность

- Всё распознавание речи работает **локально** — Parakeet TDT исполняется на устройстве через CoreML / Apple Neural Engine
- Аудио не записывается на диск и не сохраняется дольше одной сессии
- В облачный LLM уходит **только текст** — и только если вы сами настроили облачный сценарий (Anthropic / OpenAI / GLM)
- При использовании Ollama — вся обработка остаётся на вашем компьютере

## Архитектура

HUSH написан на Python 3.14 с нативным UI через PyObjC + AppKit. Никакого Electron, никаких веб-технологий — всё нативное.

```
main.py          Хоткеи, жизненный цикл сессии, вставка текста
overlay.py       Весь UI: тихий режим, полный режим, история, темы
recorder.py      Захват аудио (sounddevice)
transcriber.py   Обёртка над Parakeet TDT subprocess + CoreML warmup
processor.py     Маршрутизация LLM (Ollama / Anthropic / OpenAI / GLM)
injector.py      Вставка текста через Accessibility API
config.py        Пути, API ключи, константы
build_app.sh     Сборка HUSH.app bundle
```

## История версий

### v1.1 — 25.06.2026

**Система гридов для панелей — переписана с нуля**

- Сброс (`[🔄]`) теперь размещает все окна как группу по центру экрана. Паттерн выбирается автоматически по размеру экрана: крест 3×3 для большого, 3×2 для среднего (большинство MacBook), 2×2 для компактного.
- После сброса панели никогда не перекрываются: позиции вычисляются геометрически до открытия, затем вся группа сдвигается на экран.
- Новое правило размещения: диагональная ячейка используется только если хотя бы одна соседняя по грани уже занята — окна не соединяются «только углом».
- При перетаскивании за край экрана панель перестраивается на противоположную сторону, а не строит лишнюю 4-ю горизонтальную колонку.

**Единообразие интерфейса**

- Все кнопки приведены к единому стилю `[ПРОПИСНЫМИ]` в квадратных скобках (RU/EN/ES).
- Цвет кнопки `[СЦЕНАРИЙ]` теперь зависит от темы: `C_YEL` задаётся отдельно для каждой темы и обновляется при переключении. Раньше жёлтый был хардкоженным и плохо читался на светлых темах.

**Мониторинг провайдеров**

- Кнопки сценариев становятся красными если назначенный LLM-провайдер недоступен. Статус проверяется каждые 30 с в фоновом потоке; список сценариев обновляется автоматически при каждом изменении статуса.
